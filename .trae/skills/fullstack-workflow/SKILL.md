# 全栈开发流程验证

## 功能
1. **范围分析**
   - 确定功能涉及的技术层次
   - 识别需要修改的文件类型
   - 评估前端、后端、数据库的改动范围

2. **API对接验证**
   - 检查API接口是否存在
   - 验证请求/响应格式
   - 确认参数和返回值类型

3. **前端验证**
   - 检查组件和页面
   - 验证状态管理
   - 确认API调用方式

4. **后端验证**
   - 检查控制器/路由
   - 验证业务逻辑
   - 确认数据访问层

5. **数据库验证**
   - 检查表结构
   - 验证数据模型
   - 确认数据操作

6. **全链路测试**
   - 端到端流程验证
   - 数据流转检查
   - 错误处理验证

## 使用场景
- 当功能需要修改前端+后端+数据库时
- 当需要验证API对接是否完整时
- 当进行全栈功能开发时
- 当需要确保数据流程畅通时

## 工作流程

### 1. 需求分析
1. 理解功能需求
2. 识别涉及的实体/数据模型
3. 确定用户交互流程

### 2. 范围确定
1. **前端范围**
   - 需要修改哪些页面/组件
   - 是否需要新的API调用
   - 状态管理是否需要更新

2. **后端范围**
   - 需要创建/修改哪些API
   - 业务逻辑在哪里
   - 是否需要新的服务层

3. **数据库范围**
   - 表结构是否需要变更
   - 是否需要数据迁移
   - 索引是否需要优化

### 3. 代码分析
1. **前端代码分析**
   - 查找相关页面组件
   - 查看现有API调用方式
   - 了解状态管理模式

2. **后端代码分析**
   - 查找相关路由/控制器
   - 查看现有API实现
   - 了解数据访问模式

3. **数据库分析**
   - 查看相关表结构
   - 了解数据关系
   - 确认数据类型

### 4. 实施计划
1. 确定开发顺序
2. 列出需要创建/修改的文件
3. 规划测试步骤

### 5. 开发执行
1. **数据库层（如需要）**
   - 修改表结构
   - 添加/修改索引
   - 数据迁移

2. **后端层**
   - 创建/修改API路由
   - 实现业务逻辑
   - 添加数据验证

3. **前端层**
   - 创建/修改页面组件
   - 实现API调用
   - 更新状态管理

### 6. 流程验证
1. **API验证**
   - 测试API接口
   - 验证请求参数
   - 检查响应格式

2. **前端→后端验证**
   - 检查前端API调用
   - 验证数据传递
   - 确认错误处理

3. **后端→数据库验证**
   - 检查数据操作
   - 验证数据持久化
   - 确认事务处理

4. **端到端验证**
   - 完整流程测试
   - 边界情况测试
   - 错误场景测试

## 常见问题处理

### API对接问题
- 检查API路径是否正确
- 验证请求方法（GET/POST/PUT/DELETE）
- 确认请求头设置
- 检查CORS配置

### 数据格式问题
- 验证前后端数据类型匹配
- 检查日期/时间格式
- 确认枚举值一致
- 验证嵌套数据结构

### 状态同步问题
- 检查前端状态更新
- 验证缓存策略
- 确认实时更新机制
- 检查乐观/悲观锁

## 示例

### 场景：添加商品发布功能
1. **范围确定**
   - 前端：发布页面、表单组件、API调用
   - 后端：商品创建API、商品服务
   - 数据库：商品表（已存在）

2. **代码分析**
   - 前端：查看现有publish.vue
   - 后端：查找items相关API
   - 数据库：确认items表结构

3. **开发顺序**
   - 后端：创建商品API
   - 前端：完善发布页面
   - 测试：端到端验证

4. **验证流程**
   - API测试：POST /api/items
   - 前端测试：填写表单→提交
   - 数据库测试：检查数据是否保存
   - 完整流程：从发布到列表展示
