# 前端测试技能

## 功能
- 自动启动后端服务（端口3000）
- 自动启动前端服务（H5模式）
- 读取浏览器控制台日志
- 检查JavaScript错误和警告
- 验证API调用是否成功
- 自动修复发现的bug

## 使用场景
- 每次完成代码修改后
- 添加新功能后
- 修复bug后需要验证
- 合并代码前
- 代码审查前

## 工作流程

### 1. 启动服务
```bash
# 启动后端
cd backend && npm start

# 启动前端（H5模式）
cd frontend && npm run dev:h5
```

### 2. 等待服务启动
- 后端：等待 "Server is running on port 3000"
- 前端：等待 "ready in Xms" 和本地URL

### 3. 检查浏览器日志
打开浏览器开发者工具（F12），检查：
- Console 中的错误（红色）
- Console 中的警告（黄色）
- Network 中的API请求状态
- 页面是否正常渲染

### 4. 常见错误修复

#### 导入错误
```typescript
// 错误：从vue导入uni-app生命周期钩子
import { onShow, onLoad } from 'vue'  // ❌

// 正确：从@dcloudio/uni-app导入
import { onShow, onLoad } from '@dcloudio/uni-app'  // ✅
```

#### 重复导入
```typescript
// 错误：重复导入
import { ref, computed } from 'vue'
import { onMounted } from 'vue'  // ❌

// 正确：合并导入
import { ref, computed, onMounted } from 'vue'  // ✅
```

#### 类型错误
- 检查接口定义是否匹配
- 检查props类型是否正确
- 检查响应式数据类型

#### API错误
- 检查请求URL是否正确
- 检查请求参数格式
- 检查响应数据处理

### 5. 验证清单

#### 页面功能
- [ ] 页面正常加载
- [ ] 无JavaScript错误
- [ ] 无API请求失败
- [ ] 数据正确显示
- [ ] 交互功能正常

#### 样式检查
- [ ] 布局正确
- [ ] 响应式正常
- [ ] 无样式冲突

#### 性能检查
- [ ] 页面加载时间
- [ ] API响应时间
- [ ] 无内存泄漏

## 测试步骤

1. **启动后端服务**
   ```bash
   cd d:\work\share-camp-master\backend
   npm start
   ```

2. **启动前端服务**
   ```bash
   cd d:\work\share-camp-master\frontend
   npm run dev:h5
   ```

3. **打开浏览器**
   - 访问前端URL（通常是 http://localhost:5178/）
   - 打开开发者工具（F12）

4. **执行测试**
   - 测试修改的功能
   - 检查控制台错误
   - 检查网络请求
   - 验证数据流

5. **修复问题**
   - 记录所有错误
   - 分析错误原因
   - 实施修复
   - 重新测试

6. **验证修复**
   - 确认错误已解决
   - 检查是否有新错误
   - 验证功能正常

## 常见问题和解决方案

### 问题1：端口被占用
**解决方案**：前端会自动尝试其他端口（5174, 5175, 5176, 5177, 5178）

### 问题2：API请求失败
**解决方案**：
- 检查后端服务是否运行
- 检查请求URL是否正确
- 检查请求头（特别是Authorization）
- 检查网络连接

### 问题3：页面白屏
**解决方案**：
- 检查JavaScript错误
- 检查路由配置
- 检查组件导入
- 检查数据初始化

### 问题4：样式异常
**解决方案**：
- 检查CSS导入
- 检查类名冲突
- 检查响应式断点
- 检查浏览器兼容性

## 最佳实践

1. **每次修改后都运行测试**
2. **修复一个bug后立即测试**
3. **不要积累多个修改再测试**
4. **记录所有发现的错误**
5. **验证修复不会引入新问题**
